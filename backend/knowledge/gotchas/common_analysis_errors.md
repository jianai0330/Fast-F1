# 常见分析错误与修正指南

本文档记录F1数据分析中常见的陷阱和错误，以及对应的修正方法。这些错误可能导致AI分析输出错误结论，必须被系统性地识别和避免。

---

## 1. DRS数据在VSC/SC期间无效

**错误描述**：
VSC(虚拟安全车)或SC(安全车)期间，DRS不可用。如果将VSC/SC期间的圈时数据用于正常对比，会误认为DRS直道的速度"骤降"是性能问题。

**典型表现**：
- 某圈DRS直道最高速比前后圈低15-25km/h
- 圈时突然增加2-5s
- S3(通常含DRS直道)时间突然恶化

**修正方法**：
- 检查`track_status_data`：'1'=正常, '4'=SC, '6'=VSC, '7'=SC结束
- 排除所有非'1'状态的圈
- DRS不可用时，前车圈时数据与后车不具可比性(前车无DRS，后车也无)

**AI分析提示**：
> "VSC期间的速度下降不是性能问题，而是规则限制。分析时必须剔除VSC/SC圈号。"

## 2. 进站圈/出站圈数据需排除

**错误描述**：
进站圈(pit-in)和出站圈(pit-out)的圈时包含进站时间，不能用于性能分析。直接纳入会导致：
- 圈时比正常慢20-25s(pit loss)
- 退化斜率被严重扭曲
- 圈时标准差被虚增

**典型表现**：
- 某圈圈时突然增加20s以上
- `PitInTime` 或 `PitOutTime` 非空
- 出站圈速度分布异常(前半段极慢，后半段正常)

**修正方法**：
```python
# 排除进站圈和出站圈
valid_laps = laps[
    laps['PitInTime'].isna() & laps['PitOutTime'].isna()
]
```
- 这是 `rule_engine.py` 中 `analyze_stint_degradation` 已经实现的过滤
- 但 `analyze_tyre_stability` 中使用的是 `get_lap_times()`，未过滤pit圈——这是一个已知缺陷

**AI分析提示**：
> "进站圈/出站圈数据不可用于性能对比。已排除PitIn/PitOut圈。"

## 3. 2026 circuit_info.corners['Distance']全为NaN

**错误描述**：
在2026赛季的FastF1数据中，`circuit_info.corners['Distance']` 列全部为NaN。这导致规则引擎的 `analyze_corners()` 函数无法正确计算弯角分析窗口。

**典型表现**：
- `analyze_corners()` 返回空列表（所有弯角都被跳过）
- 遥测对标分析中缺少弯角数据
- Prompt中"弯道攻略"段落为空或仅含fallback信息

**修正方法**：
1. **方案A：手动弯角距离**：为每条赛道维护一份手动弯角距离表
2. **方案B：基于遥测推断**：通过Speed曲线的局部最小值自动检测弯角位置
3. **方案C：混合方案**：使用赛道知识库中的 `sectors.key_corners` 信息辅助推断

**代码级修正示例**：
```python
def get_corner_distances(session, fallback_track_name=None):
    """获取弯角距离，2026数据NaN时使用fallback"""
    circuit_info = session.get_circuit_info()
    corners = circuit_info.corners
    
    if corners['Distance'].isna().all():
        # Fallback: 使用预定义的赛道弯角距离
        from knowledge.tracks import get_fallback_corners
        return get_fallback_corners(fallback_track_name)
    
    return corners['Distance'].values, corners['Number'].values
```

**AI分析提示**：
> "2026赛季弯角距离数据缺失，弯角分析基于赛道知识库的近似定位，精度有限。刹车点差异和最低速差异可能存在±50m的窗口偏移。"

## 4. 双停vs单停的轮胎退化不可直接对比

**错误描述**：
不同策略类型（一停 vs 两停）的轮胎退化斜率不可直接对比，因为：
- 一停的轮胎经历了更长的温度循环（从冷到热到过热）
- 两停每段轮胎温度历史更短，峰值温度可能不同
- 燃油负载差异：一停在stint后期燃油更轻(更快)，两停每段stint初燃油更重(更慢)

**典型表现**：
- 一停车手硬胎slope=0.03s/圈，两停车手中性胎slope=0.04s/圈
- 错误结论："硬胎退化比中性胎慢"——实际是策略类型差异

**修正方法**：
- 对比退化斜率时必须标注策略类型
- 同胶质同策略类型才可直接对比
- 跨策略对比时需修正燃油负载差异

**AI分析提示**：
> "一停与两停策略的退化斜率不可直接对比。一停的硬胎经历了45圈的温度循环，而两停的中性胎每段仅20圈，温度历史完全不同。"

## 5. 燃油负载差异影响直线速度比较

**错误描述**：
正赛中燃油负载从~100kg逐渐降低到~5kg，导致：
- stint初期的直线速度比stint末期慢(额外重量拖累加速)
- 比较不同车手的直线速度时，如果处于stint不同阶段，燃油差异可达30-50kg
- 30kg燃油差异 = 约1.0s圈时差 = 约5-8km/h直线速度差

**典型表现**：
- A车手stint末期最高速340km/h，B车手stint初期最高速332km/h
- 错误结论："A的PU更强"——实际可能是A轻了40kg燃油

**修正方法**：
- 使用 `fuel_correction_factors.json` 中的赛道修正因子
- 计算公式：`corrected_speed ≈ actual_speed + (fuel_delta_kg × speed_factor)`
- 简化判断：stint前5圈 vs stint后5圈的直线速度差约5-8km/h是燃油效应

**AI分析提示**：
> "直线速度对比需修正燃油负载差异。车手A处于stint末期(轻油)，车手B处于stint初期(重油)，仅燃油差异即可解释5-8km/h的速度差。"

## 6. 天气变化导致的圈时波动

**错误描述**：
赛道温度和风速在比赛中持续变化，导致圈时波动并非纯由性能或轮胎退化引起：
- 赛道温度每变化5°C，圈时变化约0.1-0.2s
- 风向变化影响直线速度和弯中平衡
- 阵风导致单圈异常(一圈快0.3s下一圈慢0.3s)

**典型表现**：
- 某几圈圈时突然变快/变慢，与轮胎退化趋势不符
- 两位车手同时在同一圈出现异常(共同环境因素)
- 逆风/顺风导致S1快S3慢或反过来

**修正方法**：
- 检查 `weather_data` 中的温度和风速变化
- 两位车手同圈号同时异常=环境因素，非个人因素
- 顺风圈速快、逆风圈速慢，直线段差异最明显

**AI分析提示**：
> "L25-L27两位车手同时圈时下降0.2s，与赛道温度上升3°C的时间吻合。这不是轮胎改善，而是温度升高带来的抓地力提升。"

## 7. 遥测数据偶发截断的处理

**错误描述**：
FastF1的遥测数据(Telemetry)偶尔出现截断：
- 某圈的遥测数据不完整（缺少部分距离段的Speed/Throttle/Brake）
- 数据可能在某个距离点突然中断
- 导致 `analyze_corners()` 的窗口内数据不全

**典型表现**：
- 某弯角的最低速异常低（实际是缺数据导致读到的是弯前/弯后值）
- 油门开度异常（缺数据段被填充为0或NaN）
- `analyze_straights()` 返回的最高速偏低

**修正方法**：
- 检查遥测数据的完整性：`len(tel) < expected_length`
- 弯角分析时检查窗口内数据量：`len(seg) < min_points` 则跳过
- 速度数据出现0或异常低值时标记为缺失
- 对比分析时，如果某车手某弯角数据缺失，不纳入对比

**代码级修正示例**：
```python
def get_window(tel):
    window = tel[(tel['Distance'] >= win_start) & (tel['Distance'] <= win_end)]
    # 检查数据完整性
    if len(window) < 20:  # 最少20个采样点
        return pd.DataFrame()  # 返回空，触发跳过逻辑
    # 检查异常值
    if window['Speed'].min() < 50:  # 低于50km/h可能是数据错误
        window = window[window['Speed'] >= 50]
    return window
```

**AI分析提示**：
> "某车手某些弯角的遥测数据不完整，可能影响刹车点和最低速的准确性。数据缺失的弯角已在分析中标注。"

## 8. 赛道演进(Evolution)效应

**错误描述**：
赛道表面在周末期间持续演进（rubbering in），导致：
- 周五圈时比周日慢1.5-2.5s（仅因赛道改善）
- FP2的圈时不可直接与排位赛/正赛对比
- 正赛前10圈赛道仍在演进，退化分析前几圈可能被赛道改善抵消

**典型表现**：
- FP2最快圈比排位赛慢2s以上
- 正赛前5-10圈圈时持续下降（赛道演进+燃油烧耗双重效应）
- 不同时段比赛的策略不可直接对比（Sprint vs 正赛）

**修正方法**：
- 不同session的圈时不可直接对比
- 正赛stint分析中，前5-10圈的"性能改善"部分来自赛道演进
- 粗略估计赛道演进效应：每圈约0.02-0.05s改善，持续10-15圈

## 9. 安全车/VSC圈时扭曲

**错误描述**：
安全车/VSC期间所有车手减速，圈时异常，但如果不排除：
- 退化斜率被严重拉低（安全车圈慢但不是"退化改善"）
- 标准差被虚增
- stint分段时安全车圈可能被误认为是Compound变化的分界点

**典型表现**：
- 某圈圈时比中位数高10-20s（安全车圈）
- stint中间出现一次极慢圈后恢复正常
- `track_status` 为 '4' 或 '6'

**修正方法**：
```python
# rule_engine.py 已有简单过滤：圈时 > 中位数 * 1.2
# 但安全车圈可能仅慢20-30%，1.2倍阈值不够
# 更可靠的方法：检查 track_status_data
def filter_sc_laps(laps, session):
    track_status = session.track_status_data
    # 标记安全车圈
    sc_laps = identify_sc_laps(track_status, laps)
    return laps[~laps['LapNumber'].isin(sc_laps)]
```

## 10. 脏空气(Dirty Air)导致的性能偏差

**错误描述**：
跟车时前车的尾流效应导致：
- 下压力损失约20-30%
- 轮胎温度升高6-8°C（散热不良）
- 退化速率加速约30%
- 圈时增加0.3-1.0s（取决于跟车距离）

**典型表现**：
- 某车手某段stint退化异常快（实际是被前车阻挡的脏空气）
- 同一车手不同stint退化差异大（一段跟车一段自由空气）
- 排位赛(自由空气)与正赛(可能有脏空气)的遥测数据不可直接对比

**修正方法**：
- 检查 `position_data` 判断跟车距离
- 距离<1.5s的圈标记为"脏空气圈"
- 退化分析中分别统计自由空气和脏空气的slope
- 遥测对比时优先使用排位赛数据（自由空气）

**AI分析提示**：
> "车手A的stint 2退化斜率(0.07s/圈)明显高于stint 1(0.04s/圈)，但stint 2期间有8圈紧跟前车，脏空气使轮胎温度升高约7°C，退化加速约30%。修正脏空气影响后，stint 2的真实退化约0.05s/圈。"

## 错误优先级

| 优先级 | 错误 | 影响程度 | 修正难度 |
|--------|------|---------|---------|
| P0-必须修正 | 进站圈/出站圈数据 | 极大(扭曲所有统计) | 低(已实现) |
| P0-必须修正 | VSC/SC期间DRS无效 | 大(速度对比错误) | 中 |
| P1-重要 | 2026弯角距离NaN | 大(缺少弯角分析) | 高(需手动数据) |
| P1-重要 | 燃油负载差异 | 中(直线速度误判) | 中(有修正因子) |
| P2-需注意 | 双停vs单停不可比 | 中(退化对比错误) | 低(标注即可) |
| P2-需注意 | 天气变化 | 中(圈时波动误归因) | 中(需天气数据) |
| P3-了解 | 遥测截断 | 低(偶发) | 低(检查完整性) |
| P3-了解 | 赛道演进 | 低(跨session比较时) | 低(标注即可) |
| P3-了解 | 脏空气 | 低(正赛常见) | 高(需position_data) |
