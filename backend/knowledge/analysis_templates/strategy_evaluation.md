# 策略评估模板

## 概述

F1策略评估是将赛道数据、轮胎退化和比赛态势综合分析的过程。本模板定义策略评估的标准框架，确保分析从策略识别到效果评估形成完整闭环。

---

## 标准评估结构

### 1. 策略识别

**目标**：识别两位车手各自的策略类型和关键参数

**必须提取的信息**：

| 参数 | 数据来源 | 说明 |
|------|---------|------|
| 停站次数 | `key_laps` 中 reason=pit_stop | 一停/两停/三停 |
| 每段胶质 | `stint_degradation` compound | S/M/H组合 |
| 进站圈号 | `key_laps` pit事件 | L18/L35等 |
| 每段长度 | `stint_degradation` laps | 12圈/23圈等 |
| 退化斜率 | `stint_degradation` slope | 每段轮胎的退化速率 |

**策略类型分类**：

| 策略 | 典型组合 | 适用赛道 | 风险 |
|------|---------|---------|------|
| 一停 | M+H 或 H+M | 低退化赛道 | 后期轮胎cliff |
| 两停(标准) | M+M+M 或 S+M+M | 大多数赛道 | 安全但非最优 |
| 两停(攻击) | S+M+H 或 S+S+M | 高退化赛道 | 需要undercut配合 |
| 三停 | S+S+M 或 S+M+S | 极高退化/高温 | pit loss累计大 |

**输出格式**：
```
车手A策略：两停 [Soft L1-L12 → Medium L13-L32 → Medium L33-L50]
  - Stint 1: Soft 12圈, slope=0.06s/圈, 健康退场
  - Stint 2: Medium 20圈, slope=0.04s/圈, L28后加速退化
  - Stint 3: Medium 18圈, slope=0.04s/圈 (预估)

车手B策略：一停 [Medium L1-L25 → Hard L26-L50]
  - Stint 1: Medium 25圈, slope=0.04s/圈, L20后开始cliff
  - Stint 2: Hard 25圈, slope=0.03s/圈 (预估)
```

---

### 2. 策略执行评估

**目标**：评估策略是否按计划执行，偏差在哪里

**执行检查清单**：

#### 进站窗口是否正确
- **Undercut时机**：是否在对手轮胎过peak后进站？
- **Overcut时机**：是否利用了对手新胎warm-up期？
- **安全车利用**：是否在安全车窗口进站？

**评估方法**：
```
计划进站窗口: L18-L22
实际进站: L20
对手进站: L22
→ 提前2圈undercut，新胎2圈crossover期间每圈快0.5s
→ 净收益: 2圈 × 0.5s - pit_loss差 = 1.0s - 0.3s = 0.7s
```

#### 交通影响
- pit exit是否遇到慢车？(损失0.5-2.0s)
- out-lap是否被蓝旗阻挡？
- stint中期是否长时间跟车？(退化加速30%)

#### 环境适应
- 温度是否偏离预模拟？(每10°C偏差影响30-50%退化)
- 风向变化是否影响直线速度？
- 是否有雨战转换窗口？

**输出格式**：
```
策略执行评分: 7/10
✅ 进站窗口选择正确，L20 undercut成功
✅ 安全车窗口(L35)利用充分
⚠️ Stint 2被HAM阻挡3圈，圈时损失0.4s/圈
❌ Stint 3开始时温度下降8°C，硬胎无法进入工作窗口
```

---

### 3. 策略vs替代方案对比

**目标**：评估实际策略与最优替代方案的差距

**替代方案构建方法**：

```
实际策略: 两停 S+M+M, 总时间 T1
替代方案A: 一停 M+H, 总时间 T2
替代方案B: 两停 S+H+M, 总时间 T3
替代方案C: 三停 S+S+M, 总时间 T4

总时间计算:
T = Σ(stint圈时) + Σ(pit_loss) + 交通损失 + 环境修正
```

**对比维度**：

| 维度 | 实际策略 | 替代A | 替代B |
|------|---------|-------|-------|
| 总时间 | 基准 | +2.3s | -0.8s |
| 风险等级 | 中 | 高(cliff) | 低 |
| 安全车敏感度 | 中 | 高 | 低 |
| 交通风险 | 中 | 低 | 高(多一次进站) |

**输出要求**：
- 必须量化"如果不是这个策略，会快/慢多少"
- 标注替代方案的前提条件（如"如果没有安全车"）
- 现实约束：后视镜分析(Hindsight)需标注"在比赛当时信息下不可预知"

---

### 4. 关键转折点分析

**目标**：识别策略成败的转折点

**转折点类型**：

1. **策略胜手**：正确决策带来显著收益
   - 例：L20 undercut成功，净赚0.7s
   - 量化：计算实际收益vs不做决策的对比

2. **策略败手**：错误决策导致损失
   - 例：L35安全车未进站，对手免费进站追回3s
   - 量化：计算损失vs进站的对比

3. **外部变量**：非策略可控因素
   - 例：L28 VSC打乱了一停策略
   - 量化：标注为"不可控"但评估应对是否合理

**输出格式**：
```
转折点1 (L20): ✅ 策略胜手 - Undercut成功
  决策: 提前2圈进站换中性胎
  收益: 净赚0.7s赛道位置
  关键前提: 对手L22才进站(晚了2圈)

转折点2 (L35): ❌ 策略败手 - 安全车窗口错过
  决策: 未在安全车下进站
  损失: 对手免费进站追回3.0s
  原因: 刚换新胎5圈，双输局面

转折点3 (L42): ⚡ 外部变量 - VSC
  事件: VSC 3圈
  影响: 轮胎温度下降，重启后3圈暖胎
  应对评估: 合理(无更好选择)
```

---

### 5. 策略总评

**目标**：综合评估策略质量，给出可量化的评分

**评分维度**：

| 维度 | 权重 | 评分标准 |
|------|------|---------|
| 策略选择 | 30% | 是否选择了最优策略类型(一停/两停/三停) |
| 执行时机 | 25% | 进站窗口是否正确 |
| 环境适应 | 20% | 对温度/天气/安全车的应对 |
| 交通管理 | 15% | 是否有效避免/利用交通 |
| 风险管理 | 10% | 策略的风险敞口是否合理 |

**总评输出格式**：
```
策略总评: 7.5/10

✅ 优势: 两停策略在巴林高退化赛道是正确选择；L20 undercut时机精准
⚠️ 不足: L35安全车应对犹豫，损失3.0s；Stint 2被交通阻挡4圈
📊 最优替代: 两停 S+H+M (预估快0.8s，但需L18更早进站)
🎯 结论: 策略方向正确，执行有2个关键失误。如果安全车进站，可挽回3.0s，最终差距可能从4.2s缩小至1.2s——仍在DRS窗口内。
```

## 策略分析中常见的认知偏差

1. **后视镜偏差**：用赛后信息批评赛中决策。必须标注"在当时信息下是否合理"
2. **结果导向偏差**：策略结果好≠策略好（运气因素如安全车timing）
3. **单一变量偏差**：只看轮胎退化忽略了交通、温度、能量管理
4. **pit loss低估**：进站损失不只是一次性22s，还包括out-lap慢圈和暖胎圈
5. **安全车概率忽略**：某些赛道(Baku/Singapore)安全车概率>80%，策略必须考虑
