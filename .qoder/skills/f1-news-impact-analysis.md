# F1 新闻赛况影响分析技能 (News Impact Analysis)

## 元信息

- **技能ID**: f1-news-impact-analysis
- **版本**: 1.0.0
- **适用场景**: F1新闻事件的赛况影响评估、性能因果链分析、积分格局影响预判
- **前置依赖**: 新闻原文/摘要、当前积分榜数据、赛历信息
- **核心方法论**: 事件事实提取 → 性能因果链展开 → 积分格局映射 → 可验证预测

---

## 1. 三步推论法

### 1.1 概述

新闻赛况影响分析必须严格遵循三层递进推理，每层都必须显式标注，不可跳过：

```
第一层：事实层 — 新闻说了什么（不添加、不遗漏）
    ↓
第二层：因果层 — 这个事件通过什么机制影响性能
    ↓
第三层：格局层 — 对2026赛季积分格局的具体影响
```

### 1.2 第一层：事实提取

**原则**：忠实于原文，不添加、不推断、不遗漏

**提取清单**：

| 要素 | 说明 | 示例 |
|------|------|------|
| 事件主体 | 哪个车队/车手/供应商 | Mercedes-AMG |
| 事件内容 | 具体发生了什么 | PU升级：新增MGU-H替代方案 |
| 时间节点 | 何时生效 | 从Miami站开始 |
| 量化数据 | 原文给出的具体数字 | 功率提升约10kW |
| 信息来源 | 原文引述的官方/非官方 | 团队领队Wolff确认 |
| 不确定性 | 原文中的限定词 | "预计""据称""可能" |

**事实提取规则**：
1. 区分"官方确认"vs"媒体报道"vs"推测"
2. 区分"已发生"vs"计划中"vs"传闻"
3. 保留原文的限定词（"预计""可能"等），不升级为确定性表述
4. 如果原文信息不足，明确标注"原文未提及[X]"

**事实提取模板**：
```markdown
### 事实层
- **主体**：[车队/车手/供应商]
- **事件**：[具体内容]
- **时间**：[生效时间]
- **量化**：[原文数据或"原文未量化"]
- **来源可靠性**：[官方确认 / 权威媒体 / 推测]
- **不确定性**：[原文中的限定词]
- **缺失信息**：[原文未提及但分析需要的信息]
```

### 1.3 第二层：因果链展开

**原则**：从事件到性能影响，每一步都必须有物理/工程/竞技逻辑支撑

**因果链构建方法**：

```
事件 → [机制1] → 中间效果1 → [机制2] → 中间效果2 → ... → 性能影响

示例：
Mercedes PU升级(+10kW)
  → [功率增加] → 直线最高速+3km/h
    → [速度优势] → 功率赛道(Monza/Spa)单圈-0.15s
      → [排位优势] → 前排发车概率+15%
        → [位置优势] → 正赛免交通损失约0.3s/站
```

**因果链验证规则**：
1. 每一步都必须有已知的物理/工程原理支撑
2. 如果某一步是推测，标注[推测]和置信度
3. 如果某一步有多种可能路径，列出所有路径并评估可能性
4. 量化每一步的量级——避免"可能很大"或"影响不确定"

**因果链分类**：

| 类型 | 定义 | 示例 |
|------|------|------|
| 直接性能因果 | 事件直接影响车辆性能 | PU升级→功率+10kW→直线速度+3km/h |
| 间接性能因果 | 事件通过间接路径影响性能 | 工程师跳槽→技术方向变化→6个月后空力升级 |
| 条件性因果 | 仅在特定条件下影响 | 轮胎配方变更→仅在高温赛道有显著影响 |
| 累积性因果 | 单次影响小但累积显著 | 轻量化5kg→每圈0.03s→赛季累积约5s |

**因果链展开模板**：
```markdown
### 因果层

**主因果链**：
[事件] → [机制] → [效果1] → [机制] → [效果2] → 性能影响[量化]

**备选因果链**（如有）：
[事件] → [替代机制] → [替代效果] → 性能影响[量化]

**条件依赖**：
- [条件1]成立时：主因果链有效
- [条件1]不成立时：[替代结论]

**因果强度**：
- 链路1：[强/中/弱] — [原因]
```

### 1.4 第三层：积分格局映射

**原则**：量化到具体位次变化预期，而非笼统的"影响竞争格局"

**映射步骤**：

1. **当前积分格局**：引用2026赛季当前积分榜数据
2. **单站影响估算**：该事件对下一站成绩的量化影响
3. **累积影响估算**：该事件对剩余赛程的累积影响
4. **位次变化预期**：具体到哪些车手/车队的排名可能变化

**积分影响计算框架**：

```python
def estimate_points_impact(event, remaining_races, current_standings):
    """
    估算事件对积分格局的影响
    """
    # 单站影响
    per_race_impact = estimate_single_race_impact(event)
    
    # 受影响赛道数
    affected_races = count_affected_races(
        event, remaining_races
    )
    
    # 累积影响（考虑对手适应/升级）
    total_impact = 0
    for i, race in enumerate(affected_races):
        # 衰减因子：对手会逐步适应/升级
        decay = 0.95 ** i  # 每站衰减5%
        total_impact += per_race_impact * decay
    
    # 映射到位次变化
    standings_impact = apply_to_standings(
        total_impact, current_standings
    )
    
    return {
        'per_race_impact': per_race_impact,
        'affected_races': len(affected_races),
        'total_impact': total_impact,
        'standings_change': standings_impact,
    }
```

**格局映射模板**：
```markdown
### 格局层

**当前积分格局**：
- [车手A]：[XXX]分(P[X])
- [车手B]：[XXX]分(P[X])
- 差距：[XX]分

**单站影响**：预计[+/−X]分（[原因]）

**累积影响**：剩余[N]站中约[M]站受影响，预计累积[+/−XX]分

**位次变化预期**：
- [车手A]可能从P[X]→P[Y] [置信度]
- [车队C]可能从P[X]→P[Y] [置信度]

**关键假设**：
- [假设1]：[如果不成立，影响变为...]
- [假设2]：[如果不成立，影响变为...]
```

## 2. 时间认知规则（2026赛季正面教学）

### 2.1 核心时间认知

**当前赛季：2026**

这是分析所有新闻影响的时间基准。所有分析必须基于2026赛季的事实和数据。

### 2.2 正面教学 vs 负面禁止

**负面禁止（效果弱）**：
- ❌ "不要提到2025赛季"
- ❌ "不要说影响2025年冠军争夺"

**正面教学（效果好）**：
- ✅ "当前是2026赛季，分析基于2026赛季积分榜和赛历"
- ✅ "2026赛季从3月澳大利亚站开始，当前已进行到第[X]站"
- ✅ "2026赛季积分格局：[具体车手和分数]"
- ✅ "历史数据（2025及更早）仅作为对标参考"

### 2.3 2026赛季关键时间线

| 时间 | 事件 |
|------|------|
| 2026年3月 | 澳大利亚站，赛季开幕 |
| 2026年3-4月 | 早期赛季，数据有限 |
| 2026年5月 | 欧洲赛季开始，升级竞赛加速 |
| 2026年7月 | 赛季中段，积分差距开始明朗 |
| 2026年9月 | 亚洲赛季，天气变量增大 |
| 2026年11月 | 美洲赛季末段，争冠关键期 |
| 2026年12月 | Abu Dhabi收官战 |

### 2.4 历史数据使用规则

**允许**：
- 使用历史数据作为性能对标参考："相比2025年同站，差距缩小0.3s"
- 使用历史模式推测可能的发展趋势："历史上PU升级通常需要3-5站才能完全发挥"

**不允许**：
- 将历史数据当作当前事实："2025年的冠军争夺" ≠ "当前赛季格局"
- 将历史规律当作确定性预测："根据2025年数据，A一定会..."
- 混淆赛季："这会影响冠军争夺"而不指定是2026赛季

**历史数据引用格式**：
```
"作为参考，2025年[车队A]在[赛道B]使用类似升级后，[具体效果+量化]"
```

## 3. 新闻类型分类与处理

### 3.1 技术类新闻

**定义**：直接涉及车辆性能的技术升级/变更/问题

| 子类型 | 典型主题 | 影响路径 | 量化难度 |
|--------|----------|----------|----------|
| PU升级 | 功率/可靠性/效率 | 直线速度/能量部署 | 中（有基准对比） |
| 空力升级 | 底板/前翼/后翼/侧箱 | 下压力/阻力/弯速 | 中-高（赛道依赖） |
| 轮胎相关 | 配方/规格/温度窗口 | 退化率/抓地力 | 中（Pirelli有数据） |
| 可靠性问题 | PU/变速箱/悬挂 | 退赛概率/罚退 | 低（概率估算） |
| 规则变更 | 技术指令/澄清 | 合规性调整 | 高（不确定性大） |

**技术类新闻分析模板**：
```markdown
## [标题]

### 事实层
- 主体：[车队]
- 变更：[具体技术变更]
- 量级：[原文数据]
- 生效时间：[从哪站开始]

### 因果层
[变更] → [机制] → [性能效果] → [赛道适用性]

预计影响：
- 功率赛道（Monza/Spa/Baku）：±X.XXXs/圈
- 下压力赛道（Monaco/Hungaroring）：±X.XXXs/圈
- 混合赛道：±X.XXXs/圈

### 格局层
当前[车队]在积分榜P[X]，距前方[XX]分。
预计单站收益[X-X]分，累积收益[XX-XX]分（剩余[N]站中[M]站受益）。
```

### 3.2 竞技类新闻

**定义**：涉及车手/车队的竞技状态变化

| 子类型 | 典型主题 | 影响路径 | 量化难度 |
|--------|----------|----------|----------|
| 车手状态 | 伤病/心理/续约压力 | 圈时稳定性/极限表现 | 高（主观因素大） |
| 车手更换 | 替补/新人 | 圈时/经验/适应期 | 中（有参考数据） |
| 车队动态 | 内部竞争/策略偏向 | 积分分配/团队协作 | 高（政治因素） |
| 处罚 | 罚退/罚款 | 发车位置/积分损失 | 低（确定性影响） |

### 3.3 人事/商业类新闻

**定义**：涉及人员变动或商业决策

**处理原则**：必须分析"间接性能影响"

**影响路径模板**：
```
人事变动 → 技术方向变化 → [时间延迟6-18月] → 性能影响

示例：
- 首席空力师跳槽 → 空力开发方向调整 → 6-12月后空力升级路线图变化
- 技术总监离职 → 短期开发节奏放缓 → 2-4站内升级推迟
- 新赞助商 → 预算增加 → 中长期开发资源增加
```

**人事类新闻分析要点**：
1. 标注时间延迟——人事变动不会立即反映在性能上
2. 区分"短期干扰"vs"长期方向变化"
3. 如果无法建立性能因果链，标注为"间接影响，无法量化"

### 3.4 赛事/规则类新闻

**定义**：涉及赛历、规则、赛制变更

| 子类型 | 影响特征 | 分析重点 |
|--------|----------|----------|
| 赛历变更 | 改变赛道组合 | 哪些车队因此受益/受损 |
| 规则调整 | 改变技术合规边界 | 哪些设计方向受影响 |
| 赛制变更 | Sprint/罚退规则等 | 策略空间变化 |
| 天气预报 | 改变比赛条件 | 哪些车手/设定受益 |

## 4. 正确 vs 错误示例

### 4.1 技术升级分析

❌ **错误**："Mercedes PU升级会让他们更快，值得观察"
- 问题：无因果链、无量化、无赛道适用性、"值得观察"无信息量

✅ **正确**："Mercedes PU升级预计带来8-12kW功率提升，在Monza/Spa等功率赛道可转化为0.15-0.2s/圈优势。当前Hamilton落后Verstappen 23分，若在剩余4个功率赛道站均多拿5-8分，理论上可在Abu Dhabi前将差距压缩至10分以内[低置信度：实际效果需赛道验证]"
- 优点：量化因果链 + 积分映射 + 置信度 + 可验证条件

### 4.2 人事变动分析

❌ **错误**："Ferrari的首席工程师跳槽了，这是个大新闻"
- 问题：无性能影响分析，"大新闻"无信息量

✅ **正确**："Ferrari空力主管离职→短期内（2-4站）空力升级可能推迟→原计划Silverstone的底板升级v2可能延迟至夏休后。基于2026赛季前6站Ferrari空力升级节奏（每2站一次小升级），推迟1个升级周期预计影响约3-5分。长期方向影响无法评估——取决于继任者的技术路线[低置信度：人事变动的性能影响延迟大、不确定性强]"
- 优点：时间延迟 + 量化 + 区分短期/长期

### 4.3 规则变更分析

❌ **错误**："新规则会影响下压力，大家都要调整"
- 问题：谁受影响最大？调整成本多少？时间表？

✅ **正确**："FIA技术指令TD040限制底板柔性→Red Bull的高柔性底板设计需修改→预估下压力损失2-3%（约0.1-0.15s/圈，高下压力赛道影响更大）。其他车队底板设计已合规，不受直接影响。但二级效应：Red Bull被迫修改可能打开新的开发方向，历史经验（2023年TD018）显示受限车队2站内可恢复70-80%损失[中置信度：基于历史类比，但每站情况不同]"
- 优点：定向影响 + 量化 + 二级效应 + 历史类比 + 置信度

### 4.4 退赛/处罚分析

❌ **错误**："Verstappen退赛对冠军争夺影响很大"
- 问题：未量化，"很大"是主观判断

✅ **正确**："Verstappen退赛损失25分（胜站积分），当前积分从187→162，领先Norris的差距从38分缩窄至13分。剩余14站中，如果Norris此后每站多拿2分（平均），可在第18站反超。但Verstappen的退赛是偶发事故（首次退赛），预计不会形成趋势，因此差距缩窄更多是单次事件而非格局逆转[高置信度：积分损失确定，趋势判断基于可靠数据]"
- 优点：量化 + 趋势预判 + 可验证 + 区分单次vs趋势

## 5. 常见陷阱与Gotchas

### 5.1 推理陷阱

| 陷阱 | 描述 | 应对策略 |
|------|------|----------|
| 时间混淆 | 提到"2025赛季冠军争夺"等 | 严格使用"2026赛季"措辞 |
| 因果跳跃 | 跳过中间步骤直接给结论 | 每一步因果链都必须显式 |
| 过度量化 | 对不确定的影响给出精确数字 | 用区间而非点估计："5-8分"而非"6.3分" |
| 低估时间延迟 | 人事/技术变更立即反映在成绩上 | 标注时间延迟和不确定性 |
| 忽略对手反应 | 假设对手不做调整 | 考虑对手的适应/升级/策略调整 |
| 单因素归因 | 复杂结果归因于单一新闻 | 标注其他可能因素 |
| 幸存者偏差 | 只分析表现好的案例 | 考虑失败/无效的升级案例 |
| 信息不足强行推论 | 缺少关键信息时仍下结论 | 用"基于已知信息"+"需验证条件"结构 |

### 5.2 信息不足时的处理

**原则**：宁可说"不知道"也不编造

**信息不足处理框架**：

```markdown
### 基于[已知信息]的分析
[已有数据支撑的有限推论]

### 需要验证的条件
- [条件1]：如果成立，则[结论1]
- [条件2]：如果成立，则[结论2]

### 缺失信息
- [缺失1]：[为什么重要]
- [缺失2]：[为什么重要]

### 置信度
[低置信度：关键信息缺失]
```

**信息不足时的禁止行为**：
- ❌ 用通用表述填充（"可能影响竞争力"）
- ❌ 假设缺失信息的值
- ❌ 降低标准仍给高置信度
- ❌ "这还需要进一步观察"（废话，应说观察什么）

**信息不足时的必须行为**：
- ✅ 明确列出缺失什么信息
- ✅ 在不同假设下给出条件性结论
- ✅ 标注低置信度
- ✅ 指出"需要什么数据才能确认"

### 5.3 特殊新闻类型处理

**谣言/非官方消息**：
- 来源标注为"非官方"
- 因果链标注为"假设性"
- 不做积分格局映射（不确定度太高）
- 输出格式："据[来源]报道[事件]，如果属实，可能[影响]。但该信息未经官方确认[低置信度]"

**负面新闻（事故/争议）**：
- 严格基于事实，不做道德判断
- 分析性能影响时不带立场
- 处罚影响用量化分析，不评论公平性

**复合事件**：
- 拆分为独立事件分别分析
- 标注事件间的交互效应
- 避免混淆因果方向

## 6. 新闻分析输出规范

### 6.1 标准输出模板

```markdown
## [新闻标题] — 赛况影响分析

### 📋 事实层
- **主体**：[车队/车手/供应商]
- **事件**：[具体内容]
- **时间**：[生效时间]
- **量化**：[原文数据]
- **来源可靠性**：[官方确认/权威媒体/推测]

### 🔗 因果层
**主因果链**：
[事件] → [机制1] → [效果1] → [机制2] → [性能影响]

**条件依赖**：
- [条件1]成立时：[影响1]
- [条件2]成立时：[影响2]
```

## 7. 2026赛季特殊分析维度

### 7.1 新规则影响

**50/50 ICE-MGU-K功率比**：
- PU升级的影响评估需要区分ICE和MGU-K两个维度
- MGU-K升级→出弯加速+直线中段速度
- ICE升级→最高速+可靠性
- 两者的影响赛道不同

**能量回收变化**：
- 制动能量回收上限变化影响能量管理策略
- 连续弯角组后直线段的能量状态成为新变量
- 新闻分析中涉及PU升级时需标注"能量部署效率"维度

**主动空力规则**：
- 不同模式下的空力效率差异
- 赛道特性与空力模式匹配
- 规则变更对主动空力使用的限制

### 7.2 积分系统

**2026赛季积分系统**（如果与往年不同需更新）：
- 标准积分：25-18-15-12-10-8-6-4-2-1
- Sprint积分：8-7-6-5-4-3-2-1（如适用）
- 最快圈1分（需前十完赛）

### 7.3 赛历特征

**2026赛季赛历要点**：
- 三重/双重背靠背：连续比赛对可靠性压力增大
- 新赛道：无历史数据，需类比分析
- 夏休期：升级冻结窗口
- 收官战：Abu Dhabi，争冠决战
