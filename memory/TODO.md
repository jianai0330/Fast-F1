**目标：** 新闻和讨论帖双向打通，支持引用原文/AI分析发帖

**后端：**
- `GET /news/{id}/posts` — 查询关联该新闻的帖子列表

- `GET /forum/posts/{id}` 返回带 `news_id` + `news_title` + `news_source`

**前端 — 新闻详情页新增：**
- 每段 AI 分析右上角加「引用」按钮

**前端 — 发帖页新增：**

**数据流：**
```
新闻详情 → [引用这段] → 发帖页(quote+newsId) → 发帖
```

## 🗂️ 域名配置清单（买域名时一起做，提醒我！）

> 触发时机：准备买域名、配 HTTPS 的那次对话，把这块一起过完。

### Step 1 — 买域名 + 配 DNS
- 买一个域名（如 `f1app.cc`），推荐腾讯云 / 阿里云
- 在 DNS 控制台加 A 记录：`@` / `api` → `43.129.185.165`

### Step 2 — 配 HTTPS（Nginx + Let's Encrypt）
- 服务器上安装 certbot，申请证书
- Nginx 配反向代理：`https://api.yourdomain.com` → `http://127.0.0.1:8000`
- 微信公众平台 → 开发 → 开发管理 → **服务器域名** 加 `https://api.yourdomain.com`
- `miniprogram/app.js` 里的 `BASE_URL` 改为 `https://api.yourdomain.com`

### Step 3 — 配业务域名（webview 跳转原文必须）
- 微信公众平台 → 开发 → 开发管理 → **业务域名**
- 添加以下域名（每次最多添加 10 个）：
  - `https://www.racefans.net`（The Race）
  - `https://www.motorsport.com`
  - `https://www.crash.net`
  - `https://fr.f1i.com`
- 每个域名需要在对应服务器根目录放微信下发的 `.txt` 校验文件（微信会提供下载）
  - ⚠️ 注意：这些是**第三方网站**，我们放不了校验文件！
  - 实际上 webview 只有**我们自己的域名**能配业务域名
  - **解决方案：** 改为中转跳转 → 在后端加 `/redirect?url=xxx`，前端 webview 打开 `https://api.yourdomain.com/redirect?url=xxx`，后端 302 跳转到目标网址，业务域名只需配自己的 API 域名即可

### ⚠️ 关于第三方新闻域名的说明
微信业务域名要求在**目标服务器**上放校验文件，第三方新闻网站我们无法操作。
所以正确方式是后端加一个 302 跳转中转接口，业务域名只配我们自己的后端域名：
```
前端 webview → https://api.yourdomain.com/redirect?url=原文URL → 302 → 原新闻页
```
这样只需要把自己的后端域名加入业务域名白名单即可。
