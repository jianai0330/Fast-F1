# 微信小程序真机调试踩坑记录

## 1. input 组件文字颜色在真机上不生效

**现象**：开发者工具正常显示，真机打字看不见（文字和背景同色）

**原因**：微信小程序 `input` 组件在部分 Android 真机上，wxss 的 `color` 属性和 wxml 的 `color` 属性均不生效，文字颜色被系统强制覆盖

**解决方案**：把 `input` 换成 `textarea`，`textarea` 的 `color` 在真机上正常生效

```xml
<!-- ❌ 不稳定 -->
<input color="#e0e0e0" ... />

<!-- ✅ 稳定 -->
<textarea style="color:#e0e0e0;" auto-height ... />
```

## 2. 从发帖页跳注册页，注册页全黑/空白

**现象**：直接编译打开注册页正常，从发帖页跳转过去就全黑

**原因**：`forum-create` 的 `onShow` 里有逻辑：检测到没有 openid 就 `navigateBack()`。但 `navigateTo` 跳注册页时，发帖页的 `onShow` 也会触发，导致注册页刚渲染就被弹掉，页面栈混乱

```js
// ❌ 问题代码
onShow() {
  const openid = wx.getStorageSync('f1_openid') || ''
  if (!openid && this.data._loaded) {
    wx.navigateBack() // 注册页还没渲染完就被弹掉了
  }
}

// ✅ 修复：onShow 里不做跳转，只更新数据
onShow() {
  const openid = wx.getStorageSync('f1_openid') || ''
  if (openid) this.setData({ openid })
}
```

## 3. 删除 lazyCodeLoading 导致全局白屏

**现象**：所有页面啥也看不见，真机报错 `Can't find variable: __wxAppCode__`

**原因**：`app.json` 里的 `"lazyCodeLoading": "requiredComponents"` 被误删，导致代码包结构异常

**解决方案**：恢复该配置，并在开发者工具清缓存后重新上传

```json
// app.json 必须保留
"lazyCodeLoading": "requiredComponents"
```

## 4. 体验版正常 vs 指定页编译正常的区别

**规律**：
- 指定页编译 = 直接打开该页面，跳过所有跳转逻辑，适合单页调试
- 体验版/预览 = 走完整启动流程，会触发所有页面跳转逻辑

**结论**：指定页编译正常不代表真实流程正常，跳转相关的 bug 必须用预览/真机调试验证
