# F1 数据可视化微信小程序 — 产品设计文档

---

## 一、产品定位

面向 F1 普通观众和硬核车迷，提供赛后数据可视化分析。
用交互图表替代静态截图，让遥测数据"动起来"，降低硬核数据的阅读门槛。

---

## 二、MVP 范围（第一版）

> 先做历史赛事数据，实时功能在 v2 迭代。

### 2.1 支持的数据维度

| 维度 | 内容 |
|------|------|
| 遥测对比 | 两车速度 / 油门 / 刹车 / 档位，横轴对齐弯角编号 |
| 圈时分析 | 单车多圈趋势、轮胎策略叠加 |
| 排位赛结果 | Q1 / Q2 / Q3 各车手最快圈排名 |
| 积分榜 | 车手 & 车队当前积分走势 |

### 2.2 不做（MVP 阶段排除）

- 实时数据推送
- 用户登录 / 收藏
- 社区评论
- 付费功能

## 三、技术架构

```
┌─────────────────────────────────────────┐
│           微信小程序（前端）              │
│  页面框架: 原生小程序 or uni-app         │
│  图表库:   ECharts for 小程序 (ec-canvas)│
└───────────────┬─────────────────────────┘
                │ HTTPS API
┌───────────────▼─────────────────────────┐
│           后端服务（待定部署方式）         │
│  语言:     Python (FastAPI)              │
│  数据获取: FastF1                        │
│  缓存:     本地文件缓存 / Redis           │
│  数据格式: JSON（供前端 ECharts 直接消费）│
└───────────────┬─────────────────────────┘
                │
┌───────────────▼─────────────────────────┐
│           FastF1 + F1 官方 API            │
│  历史数据缓存在服务器本地                  │
│  每场比赛结束后手动触发一次数据拉取        │
└─────────────────────────────────────────┘
```

**部署方案（优先级排序，待选）**

1. 云函数（腾讯云 SCF）— 和微信小程序同生态，冷启动是问题
2. 轻量云服务器（2核4G 够用）— 最灵活，FastF1 缓存放本地
3. 本地 + 内网穿透 — 开发阶段临时用

## 四、页面结构

```
首页
 ├── 赛历 / 赛事列表（按赛季）
 │
 ├── 赛事详情页
 │    ├── Tab 1: 排位赛结果（Q1/Q2/Q3 排名）
 │    ├── Tab 2: 正赛圈时分析（轮胎策略 + 圈时走势）
 │    └── Tab 3: 遥测对比（选择两位车手 → 生成图表）
 │
 └── 积分榜页
      ├── 车手积分走势（折线图，多赛站）
      └── 车队积分走势
```

## 五、核心交互流程

### 5.1 遥测对比

```
用户进入赛事详情
  → 切换到「遥测对比」Tab
  → 选择车手 A（下拉，显示车队颜色）
  → 选择车手 B
  → 点击「生成对比」
  → 前端请求后端 API
  → 后端用 FastF1 拉取最快圈遥测数据（优先从缓存读）
  → 返回 JSON（Distance, Speed, Throttle, Brake, Gear）
  → 前端 ECharts 渲染 4 个联动子图
  → 横轴显示弯角编号（T1~T18）
  → 右上角显示两人圈时和差距
  → 底部如有数据丢包，显示灰色提示文字
```

### 5.2 圈时分析

```
用户进入赛事详情 → 正赛圈时分析 Tab
  → 默认展示所有车手圈时折线
  → 可筛选车手（多选）
  → 轮胎策略色块叠加在背景（软/中/硬/中间/全雨）
  → 点击某圈可查看该圈具体数据
```

## 六、后端 API 设计（草案）

| 接口 | 方法 | 说明 |
|------|------|------|
| `/seasons` | GET | 返回可用赛季列表 |
| `/events?year=2026` | GET | 返回赛季赛历 |
| `/qualifying?year=2026&round=4` | GET | 排位赛结果 |
| `/telemetry?year=2026&round=4&d1=ALB&d2=ALO` | GET | 遥测对比数据 |
| `/laptimes?year=2026&round=4&session=R` | GET | 正赛圈时数据 |
| `/standings?year=2026` | GET | 积分榜 |

返回格式统一为：
```json
{
  "status": "ok",
  "data": { ... },
  "note": "ANT telemetry truncated at 5379m — F1 API packet loss"
}
```

---

## 七、数据质量处理原则

- 遥测截断：原样展示，`note` 字段说明原因
- 圈时 NaT：跳过不展示，不报错
- 数据丢包提示统一用灰色小字，不影响主图阅读
